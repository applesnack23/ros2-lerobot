# ROS2 기본 개념 및 명령

이번 장부터는 ROS2를 이용한 로봇 제어를 본격적으로 시작합니다.

LEVEL1 에서는 로봇 개발에 필요한 Ubuntu와 Python의 기본 내용을 살펴보았습니다. LEVEL2 에서는 Ubuntu와 Python 환경에서 SO-ARM101의 모터를 직접 제어하고, 티칭과 Pick & Place 동작을 구현했습니다.

이러한 방식만으로도 로봇을 움직일 수 있지만, 시스템이 복잡해지면 하나의 프로그램에서 모든 기능을 처리하기 어려워집니다.

실제 로봇 개발에서는 여러 개발자가 역할을 나누어 작업합니다.

- 모터와 통신하는 Driver 개발
- 로봇 상태를 확인하는 Monitoring 개발
- 로봇의 위치를 저장하는 Teaching 개발
- 이동 경로를 계산하는 Motion Planning 개발
- Camera와 Sensor 데이터를 처리하는 Vision ㅐ발
- 로봇의 안전 상태를 관리하는 Safety 개발

각 기능을 별도의 프로그램으로 개발하면, 프로그램 사이에서 명령과 데이터를 주고받을 수 있는 공통된 통신 구조가 필요합니다. 이 역할을 담당하는 것이 미들웨어(Middleware)입니다.

---

#### ROS2 란

ROS는 `Robot Operating System`의 약자이지만 Windows나 Ubuntu와 같은 운영체제는 아닙니다.

ROS2는 Ubuntu와 같은 운영체제 위에서 실행되며, 로봇을 구성하는 여러 프로그램이 서로 통신할 수 있도록 도와주는 로봇 개발용 미들웨어이자 소프트웨어 프레임워크입니다.

```bash
Ubuntu
└── ROS2
    ├── Motor Driver
    ├── Teaching Program
    ├── Motion Planning
    ├── Camera
    └── Monitoring
```

ROS2에서는 각각의 기능을 Node라는 실행 단위로 구성하고, Node 사이에서 Topic, Service, Action 등의 통신 방식을 사용해 데이터를 주고받습니다.

예를 들어 모터 Driver가 현재 관절 위치를 발행하면 Monitoring 프로그램과 Motion Planning프로그램이 같은 데이터를 받아 사용할 수 있습니다.

이처럼 ROS2는 서로 다른 프로그램을 연결하기 위한 통신 방식과 데이터 형식을 표준화합니다. 덕분에 여러 개발자가 각자 기능을 개발하더라도 하나의 로봇 시스템으로 연결할 수 있습니다.

---

#### ROS2 를 사용하는 이유

ROS2를 사용하지 않고도 로봇을 개발할 수 있습니다. 실제 산업 현장에서는 성능, 실시간성, 안전 인증 또는 기존 시스템과의 호환성을 위해 자체 미들웨어를 사용하는 경우도 있습니다.

자체 시스템은 목적에 맞게 최적화할 수 있다는 장점이 있지만, 통신, 상태 관리, 기록, 시각화와 같은 기능을 직접 구현하고 유지해야 합니다.

반면 ROS2를 사용하면 다음과 같은 장점이 있습니다.

- Node 사이의 표준화된 통신
- 여러 개발자가 기능을 나누어 개발하기 쉬움
- 다양한 센서와 로봇 Driver 활용
- RViz를 이용한 로봇 상태 시각화
- Gazebo와 연동한 시뮬레이션
- MoveIt을 이용한 경로 계획과 충돌 회피
- ros2_control을 이용한 제어기 구성
- 실행 상태 확인과 데이터 기능 가능

ROS2는 범용 프레임워크이므로 전용 제어 시스템보다 구조가 복잡하고 추가적인 학습이 필요합니다. 그러나 이미 제공되는 다양한 도구와 패키지를 활용할 수 있어 복잡한 로봇 시스템을 효율적으로 개발할 수 있습니다.

#### 이번 장의 학습 내용

이번 장에서는 ROS2 프로그램을 바로 작성하기 전에, 프로그램들이 어떤 방식으로 연결되고 통신 하는지 먼저 살펴봅니다.

주요 학습 내용은 다음과 같습니다.

- Node: 하나의 기능을 수행하는 실행 단위
- Topic: 데이터를 지속적으로 발행하고 구독하는 통신 방식
- Service: 요청을 보내고 응답을 받는 통신 방식
- Action: 시간이 걸리는 작업의 목표와 진행 상태를 전달하는 방식
- rqt: Node와 통신 상태를 화면에서 확인하는 GUI 도구

이러한 기본 개념을 이해한 후에는 Python으로 ROS2 패키지와 Node를 만들고, 최종적으로 SO-ARM101을 ROS2 기반으로 직접 제어해 보겠습니다.
