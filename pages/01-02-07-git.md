# Git

### git

Git은 코드의 변경 이력을 관리하는 **버전 관리 시스템(Version Control System)** 입니다.

프로그램을 개발하다 보면 파일을 수정하고, 새로운 기능을 추가하고, 때로는 이전 상태로 되돌아가야 하는 경우가 발생합니다.

Git은 이러한 변경 내용을 기록하고 관리할 수 있도록 도와줍니다.

또한 여러 사람이 같은 프로젝트를 동시에 작업하거나, 여러 대의 PC에서 동일한 프로젝트를 관리할 때도 매우 유용합니다.

ROS2 개발 환경에서는 GitHub를 통해 패키지를 공유하거나 다른 사람이 만든 패키지를 다운로드하는 경우가 매우 많기 때문에 Git 사용은 사실상 필수라고 볼 수 있습니다.

GitHub는 Git 저장소를 인터넷 상에서 관리하고 공유할 수 있는 플랫폼입니다.

Git : 버전 관리 도구

GitHub : Git 저장소를 올려두는 온라인 서비스

---

**기본 흐름**

git의 기본 작업 흐름은 아래와 같습니다.

```bash
# 현재 폴더를 Git 저장소로 초기화
git init

# 변경된 파일을 Staging Area에 추가
git add 파일이름
git add .

# 변경 내용을 저장
git commit -m "커밋 메시지"

# GitHub에 업로드
git push origin main
```

간단하게 표현하면 다음과 같은 흐름입니다.

```jsx
파일 수정
↓
git add
↓
git commit
↓
git push
↓
Github 업로드
```

---

**GitHub에서 저장소 가져오기**

다른 환경에서 GitHub에 업로드된 프로젝트를 가져오려면 `git clone` 명령을 사용합니다.

예를 들어 ROS2 패키지를 다운로드하여 빌드하는 경우에는 다음과 같이 사용합니다.

```bash
cd ~/ros2_ws/src

git clone 저장소주소

cd ~/ros2_ws

colcon build
```

실제로 ROS2 패키지 설치 방법을 보면 대부분 다음과 같은 형태를 사용합니다.

```jsx
git clone https://github.com/...
```

---

**자주 사용하는 명령어**

```bash
# 현재 상태 확인
git status

# 커밋 이력 확인
git log

# 원격 저장소 변경사항 가져오기
git pull

# 특정 커밋 상태로 이동
git checkout 커밋해시
```

---

Git은 매우 강력하고 기능이 많은 도구입니다.

실제로는 Branch, Merge, Rebase, Conflict 해결 등 훨씬 많은 기능들이 존재합니다.

하지만 ROS2 학습 단계에서는 다음 정도만 익혀도 대부분의 작업을 수행할 수 있습니다.

```jsx
git clone
git add
git commit
git push
git pull
```

이 책에서도 앞으로 GitHub를 통해 예제 코드와 ROS2 패키지를 제공할 예정이며, Git을 활용하여 소스코드를 관리하게 됩니다.
