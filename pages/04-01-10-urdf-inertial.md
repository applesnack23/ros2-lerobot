# Inertial 이해

앞 장에서는 Link의 충돌 판정에 사용하는 `Collision`에 대해 알아보았습니다.

Collision을 정의하면 Gazebo나 MoveIt이 로봇의 형상을 바탕으로 충돌 여부를 계산할 수 있습니다. 하지만 로봇을 물리적으로 움직이는 시뮬레이션을 구현하려면 외형과 충돌 형상만으로는 충분하지 않습니다.

실제 세계의 모든 물체는 질량을 가지고 있으며, 움직임이 변할 때 관성의 영향을 받습니다.

예를 들어 같은 모터로 가벼운 플라스틱 팔과 무거운 금속 팔을 움직이면 서로 다른 결과가 나타납니다.

- 가벼운 팔은 비교적 쉽게 가속하고 정지합니다.
- 무거운 팔은 가속과 감속에 더 큰 힘이 필요합니다.
- 질량이 회전축에서 멀리 떨어질수록 회전시키기 어려워집니다.
- 무게중심 위치에 따라 로봇이 기울어지거나 넘어지는 특성이 달라집니다.

이러한 Link의 물리적 특성을 정의하는 요소가 `Inertial`입니다.

> **Inertial은 Link의 질량, 무게중심, 관성 모멘트를 정의합니다.**
> 

Link의 세 가지 주요 정보는 다음과 같이 구분할 수 있습니다.

```
Visual    = 화면에 보이는 외형
Collision = 충돌 계산에 사용하는 형상
Inertial  = 물리 계산에 사용하는 질량과 관성
```

---

#### Inertial이 필요한 이유

Gazebo와 같은 물리 시뮬레이터는 Inertial 정보를 이용하여 다음과 같은 내용을 계산합니다.

- 중력에 의한 낙하
- 힘에 따른 가속도
- 토크에 따른 각가속도
- 관절이 움직일 때 발생하는 관성
- 로봇의 무게중심
- 물체가 넘어지거나 회전하는 동작
- 충돌 후 물체의 움직임
- 모터가 Link를 움직일 때 필요한 힘

기본적인 병진 운동은 다음 관계를 따릅니다.

$$
F = ma
$$

- F: 물체에 작용하는 힘
- m: 물체의 질량
- a: 물체의 가속도

같은 힘을 가했을 때 질량이 클수록 가속도는 작아집니다.

회전 운동도 비슷한 관계를 가집니다.

$$
\tau = I\alpha
$$

- τ: 회전축에 작용하는 토크
- I: 관성 모멘트
- α: 각가속도

같은 토크를 가해도 관성 모멘트가 크면 회전 가속도가 작아집니다.

따라서 질량과 관성값이 실제 로봇과 다르면 시뮬레이션 속도, 진동, 정지 특성 등이 실제 로봇과 다르게 나타날 수 있습니다.

---

#### Inertial 기본 구조

Inertial의 기본 구조는 다음과 같습니다.

```xml
<link name="arm_link">
    <inertial>
        <origin xyz="0 0 0" rpy="0 0 0"/>

        <mass value="1.0"/>

        <inertia
            ixx="0.01"
            ixy="0.0"
            ixz="0.0"
            iyy="0.01"
            iyz="0.0"
            izz="0.01"/>
    </inertial>
</link>
```

Inertial은 세 가지 핵심 요소로 구성됩니다.

```
inertial
├── origin
├── mass
└── inertia
```

| 요소 | 역할 |
| --- | --- |
| `<origin>` | 질량 중심의 위치와 관성 좌표계 방향 |
| `<mass>` | Link의 질량 |
| `<inertia>` | Link의 회전 관성 |

---

#### Mass

`mass`는 Link의 질량을 정의합니다.

```xml
<mass value="1.5"/>
```

URDF에서 질량의 기본 단위는 킬로그램입니다.

```
0.001 kg = 1 g
0.1 kg   = 100 g
0.5 kg   = 500 g
1.0 kg   = 1 kg
5.0 kg   = 5 kg
```

예를 들어 다음 값은 147 g을 의미합니다.

```xml
<mass value="0.147"/>
```

질량은 반드시 0보다 큰 값을 사용해야 합니다.

질량을 0으로 설정하거나 실제보다 지나치게 작게 설정하면 물리 엔진의 계산이 불안정해질 수 있습니다.

---

#### Inertial Origin

Inertial의 Origin은 Link Frame을 기준으로 질량 중심과 관성 좌표계의 방향을 정의합니다.

```xml
<origin xyz="0 0 0.05" rpy="0 0 0"/>
```

위 설정은 질량 중심이 Link Frame에서 Z축 방향으로 0.05 m 떨어진 곳에 있다는 의미입니다.

```
Link Frame
    ↑ 0.05 m
질량 중심
```

질량 중심은 일반적으로 다음과 같이 표현합니다.

```
Center of Mass = COM
```

Inertial Origin은 Visual Origin이나 Collision Origin과 목적이 다릅니다.

| Origin 종류 | 역할 |
| --- | --- |
| Joint Origin | Parent Link를 기준으로 Child Link 배치 |
| Visual Origin | Link Frame을 기준으로 외형 배치 |
| Collision Origin | Link Frame을 기준으로 충돌 형상 배치 |
| Inertial Origin | Link Frame을 기준으로 질량 중심과 관성축 배치 |

같은 Link 안에서도 네 가지 Origin은 서로 다른 값을 가질 수 있습니다.

---

#### 질량 중심이 중요한 이유

물체의 질량이 같아도 질량 중심의 위치에 따라 움직임이 달라집니다.

예를 들어 로봇 팔의 질량이 관절 가까이에 모여 있으면 비교적 쉽게 회전시킬 수 있습니다. 반대로 같은 질량이 팔 끝에 집중되어 있으면 더 큰 토크가 필요합니다.

```
관절 가까이에 질량 집중
→ 회전시키기 쉬움
→ 필요한 토크 감소

관절에서 멀리 질량 집중
→ 회전시키기 어려움
→ 필요한 토크 증가
```

질량 중심이 잘못 설정되면 Gazebo에서 다음과 같은 문제가 발생할 수 있습니다.

- 로봇이 예상하지 못한 방향으로 기울어짐
- 정지 상태에서도 관절이 내려감
- 로봇이 쉽게 넘어짐
- 실제와 다른 토크가 필요함
- 진동 또는 불안정한 움직임 발생

---

#### Inertial Origin의 회전

Inertial Origin에는 위치뿐 아니라 회전도 정의할 수 있습니다.

```xml
<origin xyz="0 0 0.05" rpy="0 1.5708 0"/>
```

`rpy`는 관성 텐서가 정의된 좌표계의 방향을 나타냅니다.

CAD 프로그램에서 가져온 관성값의 기준축이 URDF의 Link Frame과 다르다면 Inertial Origin의 `rpy`를 사용하여 관성 좌표계 방향을 맞출 수 있습니다.

단순하고 대칭적인 모델에서는 일반적으로 다음과 같이 사용합니다.

```xml
<origin xyz="0 0 0.05" rpy="0 0 0"/>
```

---

#### Inertia

`inertia`는 Link의 회전 관성을 정의합니다.

```xml
<inertia
    ixx="0.01"
    ixy="0.0"
    ixz="0.0"
    iyy="0.01"
    iyz="0.0"
    izz="0.01"/>
```

회전 관성은 물체가 회전 상태를 얼마나 쉽게 바꾸는지를 나타냅니다.

쉽게 표현하면 다음과 같습니다.

> **관성 모멘트는 물체가 회전 운동의 변화를 얼마나 방해하는지를 나타내는 값입니다.**
> 

질량이 크거나 질량이 회전축에서 멀리 분포할수록 관성 모멘트가 커집니다.

예를 들어 회전 의자에 앉아 팔을 벌린 상태로 회전하면 천천히 돌고, 팔을 몸 안쪽으로 당기면 빠르게 회전합니다.

전체 질량은 같지만 질량이 회전축에 가까워지면서 관성 모멘트가 작아졌기 때문입니다.

---

#### Inertia 단위

관성 모멘트의 단위는 다음과 같습니다.

```
kg·m²
```

따라서 CAD 프로그램에서 관성값을 가져올 때 단위가 `kg·mm²`, `g·mm²` 등으로 되어 있다면 URDF에 입력하기 전에 `kg·m²` 단위로 변환해야 합니다.

단위를 잘못 입력하면 값이 수천 배 또는 수백만 배 차이 날 수 있으므로 주의해야 합니다.

---

#### Inertia Matrix

회전 관성은 3×3 행렬로 표현됩니다.

$$
I =
\begin{bmatrix}
I_{xx} & I_{xy} & I_{xz} \\
I_{xy} & I_{yy} & I_{yz} \\
I_{xz} & I_{yz} & I_{zz}
\end{bmatrix}
$$

URDF에서는 행렬의 대칭 구조를 이용하여 여섯 개의 값만 작성합니다.

```xml
<inertia
    ixx="..."
    ixy="..."
    ixz="..."
    iyy="..."
    iyz="..."
    izz="..."/>
```

각 값의 의미는 다음과 같습니다.

| 값 | 의미 |
| --- | --- |
| `ixx` | X축 기준 회전 관성 |
| `iyy` | Y축 기준 회전 관성 |
| `izz` | Z축 기준 회전 관성 |
| `ixy` | X축과 Y축 사이의 관성곱 |
| `ixz` | X축과 Z축 사이의 관성곱 |
| `iyz` | Y축과 Z축 사이의 관성곱 |

---

#### 대각 성분과 비대각 성분

`ixx`, `iyy`, `izz`는 주대각 성분으로 각 축을 기준으로 한 회전 관성을 나타냅니다.

```
ixx = X축을 중심으로 회전할 때의 관성
iyy = Y축을 중심으로 회전할 때의 관성
izz = Z축을 중심으로 회전할 때의 관성
```

`ixy`, `ixz`, `iyz`는 관성곱이라고 부르며, 물체의 질량 분포가 좌표축에 대해 얼마나 비대칭인지를 나타냅니다.

질량 중심을 기준으로 하고 관성 좌표계가 물체의 주축과 일치하면 관성곱은 0이 됩니다.

```xml
ixy="0.0"
ixz="0.0"
iyz="0.0"
```

Box, Cylinder, Sphere와 같이 대칭적인 기본 형상을 좌표축에 맞춰 배치한 경우에는 관성곱을 0으로 설정할 수 있습니다.

하지만 실제 로봇의 복잡한 Link는 비대칭 구조일 수 있으므로 CAD에서 계산된 관성곱이 0이 아닐 수 있습니다. 이때 계산값을 임의로 0으로 변경하면 물리 특성이 부정확해질 수 있습니다.

---

#### 유효한 Inertia 값

관성값은 단순히 아무 양수나 입력하면 되는 것이 아닙니다.

기본적으로 다음 조건을 만족해야 합니다.

```
ixx > 0
iyy > 0
izz > 0
```

또한 물리적으로 유효한 관성 텐서가 되어야 합니다. 대표적으로 다음 관계를 만족해야 합니다.

$$
I_{xx} \leq I_{yy} + I_{zz}
$$

$$
I_{yy} \leq I_{xx} + I_{zz}
$$

$$
I_{zz} \leq I_{xx} + I_{yy}
$$

이러한 조건을 만족하지 않으면 Gazebo의 물리 엔진이 경고를 출력하거나 시뮬레이션이 불안정해질 수 있습니다.

초보 단계에서는 임의로 관성값을 작성하기보다 CAD 프로그램이나 검증된 계산식을 사용하는 것이 좋습니다.

---

#### Box의 관성 모멘트

질량이 mmm이고 X, Y, Z축 방향 길이가 각각 xxx, yyy, zzz인 Box의 관성 모멘트는 다음과 같이 계산할 수 있습니다.

$$
I_{xx} = \frac{1}{12}m(y^2+z^2)
$$

$$
I_{yy} = \frac{1}{12}m(x^2+z^2)
$$

예를 들어 다음 Box가 있다고 가정해 보겠습니다.

```
질량 = 1 kg
X 크기 = 0.2 m
Y 크기 = 0.1 m
Z 크기 = 0.05 m
```

계산 결과는 다음과 같습니다.

$$
I_{xx} = \frac{1}{12}(0.1^2+0.05^2) \approx 0.00104
$$

$$
I_{yy} = \frac{1}{12}(0.2^2+0.05^2) \approx 0.00354
$$

$$
I_{zz} = \frac{1}{12}(0.2^2+0.1^2) \approx 0.00417
$$

URDF에는 다음과 같이 작성할 수 있습니다.

```xml
<inertial>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <mass value="1.0"/>

    <inertia
        ixx="0.00104"
        ixy="0.0"
        ixz="0.0"
        iyy="0.00354"
        iyz="0.0"
        izz="0.00417"/>
</inertial>
```

이 계산은 질량 중심이 Box의 중심에 있고, Box의 축과 Inertial Frame의 축이 일치한다는 조건을 기준으로 합니다.

---

#### Cylinder의 관성 모멘트

질량이 mmm, 반지름이 rrr, 길이가 lll인 Cylinder가 Z축 방향으로 배치되어 있다면 다음과 같이 계산할 수 있습니다.

$$
I_{xx} = I_{yy} = \frac{1}{12}m(3r^2+l^2)
$$

$$
I_{zz} = \frac{1}{2}mr^2
$$

Cylinder의 길이 방향이 다른 축을 향한다면 관성축도 함께 변경해야 합니다.

---

#### Sphere의 관성 모멘트

질량이 mmm, 반지름이 rrr인 속이 꽉 찬 Sphere는 다음과 같이 계산할 수 있습니다.

$$
I_{xx} = I_{yy} = I_{zz} = \frac{2}{5}mr^2
$$

Sphere는 모든 방향에서 대칭이므로 세 축의 관성 모멘트가 같습니다.

#### Inertial 전체 예제

다음은 질량이 0.5 kg인 `arm_link`의 Inertial 예제입니다.

```xml
<link name="arm_link">
    <inertial>
        <origin xyz="0 0 0.1" rpy="0 0 0"/>

        <mass value="0.5"/>

        <inertia
            ixx="0.002"
            ixy="0.0"
            ixz="0.0"
            iyy="0.003"
            iyz="0.0"
            izz="0.001"/>
    </inertial>
</link>
```

이 설정은 다음과 같은 의미를 가집니다.

- Link 질량: 0.5 kg
- 질량 중심: Link Frame에서 Z축 방향 0.1 m
- X축 회전 관성: 0.002 kg·m²
- Y축 회전 관성: 0.003 kg·m²
- Z축 회전 관성: 0.001 kg·m²
- 관성곱: 모두 0

위 관성값은 구조 설명을 위한 예시입니다. 실제 SO-ARM101 모델에는 CAD에서 계산한 값을 사용하는 것이 좋습니다.

---

#### Visual, Collision, Inertial 전체 예제

하나의 Link에 Visual, Collision, Inertial을 모두 정의하면 다음과 같습니다.

```xml
<link name="arm_link">
    <visual>
        <origin xyz="0 0 0.1" rpy="0 0 0"/>

        <geometry>
            <mesh
                filename="package://so_arm101_description/meshes/arm.stl"
                scale="0.001 0.001 0.001"/>
        </geometry>

        <material name="gray">
            <color rgba="0.6 0.6 0.6 1"/>
        </material>
    </visual>

    <collision>
        <origin xyz="0 0 0.1" rpy="0 0 0"/>

        <geometry>
            <cylinder radius="0.035" length="0.2"/>
        </geometry>
    </collision>

    <inertial>
        <origin xyz="0 0 0.1" rpy="0 0 0"/>

        <mass value="0.5"/>

        <inertia
            ixx="0.0018"
            ixy="0.0"
            ixz="0.0"
            iyy="0.0018"
            iyz="0.0"
            izz="0.0002"/>
    </inertial>
</link>
```

각 요소의 역할은 다음과 같습니다.

| 요소 | 정의 내용 |
| --- | --- |
| Visual | 화면에 보이는 실제 외형 |
| Collision | 충돌 계산용 단순 형상 |
| Inertial | 질량, 무게중심, 회전 관성 |

Visual, Collision, Inertial의 Origin 값이 항상 같아야 하는 것은 아닙니다.

- Visual Origin은 Mesh의 모델링 원점을 보정합니다.
- Collision Origin은 충돌 형상의 중심을 배치합니다.
- Inertial Origin은 실제 질량 중심을 배치합니다.

각 정보의 기준과 목적에 맞게 값을 설정해야 합니다.

---

#### CAD 프로그램을 이용한 Inertial 계산

복잡한 Link의 질량과 관성 모멘트를 직접 계산하는 것은 어렵습니다.

실제 로봇 모델에서는 일반적으로 다음과 같은 CAD 프로그램을 사용합니다.

- Fusion 360
- SolidWorks
- Onshape
- Inventor
- Creo

CAD 프로그램은 다음 정보를 자동으로 계산할 수 있습니다.

- 부품 질량
- 질량 중심
- 관성 모멘트
- 관성곱
- 주 관성축

정확한 값을 얻으려면 CAD 모델에 실제 재질을 지정해야 합니다.

예를 들어 같은 크기의 부품이라도 재질에 따라 질량이 달라집니다.

```
플라스틱
알루미늄
철
스테인리스
```

재질의 밀도가 달라지면 질량과 관성값도 함께 달라집니다.

CAD에서 관성값을 가져올 때는 다음을 확인해야 합니다.

- 질량 단위가 kg인지
- 길이 단위가 m인지
- 관성 단위가 kg·m²인지
- 관성값이 질량 중심 기준인지
- CAD 좌표축과 URDF 좌표축이 같은지
- 제품 전체가 아닌 해당 Link만 계산했는지

CAD에서 출력된 관성값의 기준점이 질량 중심이 아니라면 URDF에 그대로 입력해서는 안 됩니다. 필요한 경우 평행축 정리를 사용해 질량 중심 기준의 관성값으로 변환해야 합니다.

---

#### 평행축 정리

관성값의 기준점을 이동해야 할 때는 평행축 정리를 사용할 수 있습니다.

간단한 형태는 다음과 같습니다.

$$
I = I_{COM} + md^2
$$

- I: 새로운 축을 기준으로 한 관성 모멘트
- ICOM: 질량 중심을 지나는 축의 관성 모멘트
- m: 질량
- d: 두 축 사이의 거리

하지만 실제 3차원 관성 텐서를 변환하려면 행렬 계산이 필요합니다. 따라서 가능하면 CAD에서 질량 중심 기준의 관성 텐서를 직접 출력하는 것이 안전합니다.

---

#### Gazebo에서 Inertial의 역할

Gazebo는 Inertial 정보를 이용하여 로봇의 동역학을 계산합니다.

대표적으로 다음과 같은 계산에 사용됩니다.

- 중력
- 가속과 감속
- 회전 운동
- 관절에 필요한 토크
- 충돌 후 반응
- 로봇의 균형
- Link의 진동
- 물체를 들었을 때의 하중 변화

Inertial 정보가 없거나 잘못되면 다음과 같은 문제가 발생할 수 있습니다.

- Link가 비정상적으로 움직임
- 로봇이 심하게 떨림
- 관절이 갑자기 튀는 현상 발생
- 로봇이 예상과 다르게 넘어짐
- 모터 명령에 지나치게 빠르게 반응함
- 작은 힘에도 Link가 크게 움직임
- 물리 엔진의 계산이 불안정해짐
- 로봇이 화면 밖으로 튀어나가는 현상 발생

시뮬레이션에서 모델이 폭발하듯 흩어지는 현상은 다음과 같은 원인으로 발생할 수 있습니다.

- 질량이 지나치게 작음
- 관성값이 지나치게 작거나 잘못됨
- Collision 형상이 서로 겹쳐 있음
- Joint Limit가 잘못됨
- Controller Gain이 지나치게 큼
- 시뮬레이션 시간 간격이 적절하지 않음

따라서 Inertial만 확인할 것이 아니라 Collision, Joint, Controller 설정도 함께 확인해야 합니다.

---

#### RViz2와 Inertial

RViz2는 물리 시뮬레이터가 아니라 시각화 도구입니다.

따라서 Inertial 정보를 이용하여 중력이나 가속도, 토크를 계산하지 않습니다.

```
RViz2
- 로봇 외형 표시
- TF 관계 표시
- JointState 반영
- 충돌 형상 확인

Gazebo
- 로봇 외형 표시
- 충돌 계산
- 질량과 관성 계산
- 중력 및 물리 계산
```

이를 간단하게 정리하면 다음과 같습니다.

```
RViz2  = 구조와 상태의 시각화
Gazebo = 물리적 움직임의 시뮬레이션
```

RViz2에서 로봇이 정상적으로 보인다고 해서 Inertial 정보가 올바르게 설정된 것은 아닙니다. Inertial의 정확성은 Gazebo에서 직접 움직여 보며 확인해야 합니다.

---

#### Inertial 확인 시 주의사항

Inertial 정보를 작성하거나 검토할 때는 다음 항목을 확인해야 합니다.

- 질량 단위가 kg인지 확인
- 관성 단위가 kg·m²인지 확인
- 질량이 0보다 큰지 확인
- 관성 대각 성분이 0보다 큰지 확인
- 질량 중심 위치가 올바른지 확인
- 관성값이 질량 중심 기준인지 확인
- 관성 좌표축이 URDF 좌표축과 일치하는지 확인
- CAD 모델에 실제 재질이 적용되어 있는지 확인
- 너무 작거나 비현실적인 값을 사용하지 않았는지 확인
- Gazebo에서 진동이나 불안정 현상이 없는지 확인

---

#### 정리

Inertial은 Link의 물리적 특성을 정의하는 요소입니다.

- `<origin>`: 질량 중심과 관성 좌표계
- `<mass>`: Link의 질량
- `<inertia>`: 회전에 대한 관성
- `ixx`, `iyy`, `izz`: 각 축의 회전 관성
- `ixy`, `ixz`, `iyz`: 축 사이의 관성곱
- 질량 단위: kg
- 관성 단위: kg·m²

가장 중요한 관계는 다음과 같습니다.

> **Visual은 외형, Collision은 충돌 형상, Inertial은 물리적 특성을 정의합니다.**
> 

URDF로 단순히 로봇을 시각화하는 경우에는 Visual과 Joint 구조만으로도 확인할 수 있습니다. 하지만 Gazebo에서 현실적인 물리 시뮬레이션을 수행하려면 Collision과 정확한 Inertial 정보가 필요합니다.
