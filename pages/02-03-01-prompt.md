# 프롬프트

이제는 단순한 Motor 제어를 넘어 LeRobot을 활용하여 Pick & Place Program을 만들어 보겠습니다.

본격적으로 Program을 작성하기 전에 이번 과정에서 활용할 프롬프트(Prompt)에 대해 간단히 알아보겠습니다.

프롬프트는 쉽게 말하면 다음과 같습니다.

> AI에게 원하는 작업과 조건을 전달하는 입력문장입니다.
> 

예를 들어 다음 문장도 하나의 프롬프트입니다.

> 고양이 그림 그려줘.
> 

AI는 이 문장에서 다음 정보를 확인합니다.

- 수행할 적업: 그림 생성
- 그림의 대상: 고양이

프로그래밍 관점에서 보면 프롬프트는 함수에 입력값을 전달하는 과정과 비슷합니다.

```python
print(2 + 3)
```

위 코드에서는 `2 + 3`을 `print()` 함수에 전달하고 그 결과로 `5`가 출력됩니다. 이와 비슷하게 AI에 프롬프트를 입력하면 AI가 내용을 처리하여 결과를 생성합니다.

다만 일반적인 Program과 AI는 동작 방식이 다릅니다.

Program은 정해진 문법과 규칙에 따라 동작하므로 문법에 오류가 있으면 실행되지 않거나 Error가 발생합니다. 반면 AI는 프롬프트가 불완전하더라도 결과를 생성할 수 있지만, 사용자의 의도와 다른 결과나 잘못된 정보를 만들어낼 수도 있습니다.

이처럼 AI가 사실과 다른 내용을 생성하는 현상을 할루시네이션(Hallucination)이라고 합니다.

따라서 AI가 생성한 결과는 항상 정확하다고 생각해서는 안되며, 특히 실제 Robot을 제어하는 Source Code는 사용자가 직접 내용을 검토하고 동작을 확인해야 합니다.

정확한 결과를 얻으려면 원하는 작업과 조건을 최대한 구체적으로 전달하는 것이 좋습니다.

두 프롬프트는 모두 고양이 그림을 요청하지만, 두 번째 프롬프트에는 고양이의 색상, 위치와 행동이 포함되어 있으므로 사용자가 원하는 결과를 더 정확하게 전달할 수 있습니다.

Robot Program을 요청할 때도 마찬가지입니다. 단순히 “Robot 제어 Program을 만들어줘” 라고 요청하기보다 하기의 예 처럼 자세한 내용을 전달하는 것이 좋습니다.

- 사용하는 Robot 과 Motor
- Motor 이름과 ID
- Serial Port
- 입력할 Key
- 저장할 데이터
- Robot의 동작 순서
- 파일 저장 형식
- 종료 조건

이 교재는 Python 전문 교재가 아니므로 모든 문법과 Source Code 작성 방법을 깊게 다루지는 않습니다. 대신 기존 Source Code를 이해하고, AI를 활용하여 필요한 기능을 단계적으로 추가하는 방식으로 실습을 진행합니다.

AI가 생성한 Source Code를 처음부터 모두 이해하서나 외울 필요는 없습니다. 우선 다음 내용을 파악하는 것이 중요합니다.

- 이 Program이 전체적으로 어떤 작업을 수행하는가
- 사용자가 입력하거나 수정하는 값은 무엇인가
- 어떤 순서로 동작하는가
- 문제가 발생하면 어디부터 확인해야 하는가

AI가 한 번에 완전한 Program을 만들어주는 경우도 있지만, 대부분은 실행과 수정을 반복하면서 원하는 결과에 가까워집니다.

이번 실습에서도 앞에서 작성한 Motor 제어 Program을 기반으로 다음과 같이 기능을 단계적으로 확장하겠습니다.

```python
Motor 제어
→ Teaching Point 저장
→ 저장된 위치로 이동
→ Automatic Pick & Place
```

이제 AI에 구체적인 프롬프트를 입력하고, 생성된 Source Code를 실행하고 수정하면서 Teaching과 Pick & Place Program을 만들어 보겠습니다.
